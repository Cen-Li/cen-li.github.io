Lab 19 - Stack Application

In this lab, we use the STL container "Stack" for an application. The characteristic of a Stack is : "Last In First Out".

The STL stack method include:

  • pop: Remove the top element
  • push: Insert the next element
  • top: Access the next element
  • size: Return size
  • empty: Return whether the container is empty

Write a program that will allow the user to evaluate postfix expressions. A postfix expression is an arithmetic expression where the operator occurs after the expression. For example, using infix notation we would add 4 and 5 using 4 + 5. In postfix notation we would arrange the operator to occur at the end: 4 5 +. A postfix expression can become arbitrarily complex such as 3 4 5 + - 2 *. This expression would be expressed using infix notation like this: (3 - (4+5)) * 2. As you can see, the postfix form does not require parentheses because the order of operations is made explicit. It turns out that evaluating postfix expressions is quite easy when combined with a stack data structure.

We read a postfix expression from left to right pushing values onto the stack when a number is encountered and popping numbers from the stack when an operator is encountered. For example, the expression 4 5 + would require the following operations:

  1. Push 4 onto the stack
  2. Push 5 onto the stack
  3. Pop the top two values of the stack, add them, and then push the result back onto the stack.
    1. first pop operation will pop operand 2, i.e., op2 is 5
    2. first pop operation will pop operand 1, i.e., op1 is 4
    3. apply the operator between these two operands in the sequence: op1 + op2

Your program should support the basic arithmetic binary operators such as +, -, *, and / for addition, subtraction, multiplication, and division. You can test your program with these postfix and their expected results:

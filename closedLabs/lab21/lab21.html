Lab 21 -- Circular Doubly Sorted Linked List with Dummy Head

In this lab, we will practice coding one type of linked list variation: the Circular Doubly Linked List with Dummy Head. The list is to be maintained in sorted (ascending) order. The advantages of this linked list variation include :

  1. When coding for "insertion" and "deletion", there is no need to differentiate between the two cases: (1) insert/delete at the beginning of the list; (2) insert/delete at the middle or end of the list
  2. For maintaining a sorted list, only one temporary pointer "cur" is needed to locate the insertion/deletion location and to perform the insertion/deletion operation.

This program reads in a number of string type values from a data file and inserts the items into the list maintaining the sorted order of the list after each item is inserted. The format of the data file is as the following:

apple pineapple orange milk ... Your program will read the items frwm the data file one at a time and inserts the item into the list. It repeats these read-insert steps til the end of the data file is reached. The content of the sorted list after each read-insert step is as following:
  • When the list is first created, an empty circular doubly linked list with dummy head is: head<->dummy-head-node (and dummy->next points to dummy; dummy->precede points to dummy.)
  • After the first item "apple" is read, the list content is head<->dummy<->apple<->dummy. (apple->next points to dummy, dummy->precede points to apple)
  • After the item "pineapple" is read, the list is : head<->dummy<->apple<->pineapple<->dummy. (pineapple->next points to dummy, dummy->precede points to pineapple)
  • After the item "orange" is read, the list is : head<->dummy<->apple<->orange<->pineapple<->dummy.
  • After the item "milk" is read, the list is : head<->dummy<->apple<->milk<->orange<->pineapple<->dummy.
